### 1.2 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **移动端** | Kotlin (Android) / Swift (iOS) | 原生开发，性能最优 |
| **后端** | Node.js + TypeScript / Go | 高并发，开发效率高 |
| **API网关** | Kong / AWS API Gateway | 限流、认证、日志 |
| **主数据库** | PostgreSQL 15+ | 关系型，ACID保证 |
| **缓存** | Redis 7+ | 高性能缓存，会话管理 |
| **搜索** | Elasticsearch 8+ | 全文搜索，地理搜索 |
| **向量数据库** | Pinecone / Milvus | AI嵌入向量存储 |
| **对象存储** | MinIO / AWS S3 | 图片、视频存储 |
| **消息队列** | RabbitMQ / Kafka | 异步任务处理 |
| **容器化** | Docker + Kubernetes | 弹性伸缩 |

### 2.2 数据表详细设计

#### 2.2.1 用户表 (users)

```sql
CREATE TABLE users (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email           VARCHAR(255) UNIQUE NOT NULL,
    password_hash   VARCHAR(255) NOT NULL,
    nickname        VARCHAR(50) NOT NULL,
    avatar          VARCHAR(500),
    phone           VARCHAR(20),
    gender          VARCHAR(10),
    birthday        DATE,
    bio             TEXT,

    -- 会员相关
    membership_type VARCHAR(20) DEFAULT 'free',  -- free, explorer, pro
    membership_expire_at TIMESTAMP,

    -- 统计数据
    contribution_points INTEGER DEFAULT 0,
    level           INTEGER DEFAULT 1,

    -- 偏好设置
    preferences     JSONB DEFAULT '{}',  -- 用户偏好标签

    -- 状态
    status          VARCHAR(20) DEFAULT 'active',  -- active, banned, deleted
    email_verified  BOOLEAN DEFAULT FALSE,

    -- 时间戳
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login_at   TIMESTAMP
);

-- 索引
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_membership ON users(membership_type, membership_expire_at);
CREATE INDEX idx_users_location ON users USING GIST(location);

-- 注释
COMMENT ON TABLE users IS '用户基础信息表';
COMMENT ON COLUMN users.preferences IS '用户偏好，JSON格式: {"tags": ["自然", "摄影"], "budget": "medium"}';
```

#### 2.2.2 景点表 (spots)

```sql
CREATE TABLE spots (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- 基本信息
    name            VARCHAR(200) NOT NULL,
    name_en         VARCHAR(200),
    description     TEXT,
    ai_summary      TEXT,  -- AI生成的简短描述

    -- 位置信息
    country         VARCHAR(100) NOT NULL,
    province        VARCHAR(100),
    city            VARCHAR(100),
    district        VARCHAR(100),
    address         VARCHAR(500),
    location        GEOGRAPHY(POINT, 4326),  -- PostGIS地理坐标

    -- 分类与标签
    category_id     UUID REFERENCES categories(id),
    tags            VARCHAR(50)[],  -- 标签数组
    themes          VARCHAR(50)[],  -- 主题数组

    -- 评分与热度
    rating          DECIMAL(3,2) DEFAULT 0.0,
    review_count    INTEGER DEFAULT 0,
    view_count      INTEGER DEFAULT 0,
    favorite_count  INTEGER DEFAULT 0,

    -- 人流信息
    crowd_level     VARCHAR(20) DEFAULT 'low',  -- low, medium, high
    crowd_data      JSONB,  -- 实时/历史人流数据

    -- 访问信息
    opening_hours   JSONB,  -- 营业时间
    ticket_price    DECIMAL(10,2),
    ticket_info     TEXT,
    suggested_duration INTEGER,  -- 建议游玩时长(分钟)

    -- 最佳访问时间
    best_seasons    VARCHAR(20)[],
    best_time_of_day VARCHAR(50)[],  -- morning, afternoon, evening

    -- 媒体资源
    cover_image     VARCHAR(500),
    images          JSONB,  -- 图片列表
    video_url       VARCHAR(500),

    -- 来源与验证
    source          VARCHAR(50),  -- manual, crawl, user_submit
    source_url      VARCHAR(500),
    verified        BOOLEAN DEFAULT FALSE,
    verifier_id     UUID REFERENCES users(id),

    -- 状态
    status          VARCHAR(20) DEFAULT 'active',  -- active, hidden, deleted

    -- 时间戳
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    published_at    TIMESTAMP
);

-- 索引
CREATE INDEX idx_spots_location ON spots USING GIST(location);
CREATE INDEX idx_spots_category ON spots(category_id);
CREATE INDEX idx_spots_tags ON spots USING GIN(tags);
CREATE INDEX idx_spots_rating ON spots(rating DESC);
CREATE INDEX idx_spots_crowd ON spots(crowd_level);
CREATE INDEX idx_spots_city ON spots(country, province, city);
CREATE INDEX idx_spots_status ON spots(status, published_at);

-- 全文搜索索引
CREATE INDEX idx_spots_search ON spots USING GIN(
    to_tsvector('simple', name || ' ' || COALESCE(description, ''))
);

COMMENT ON TABLE spots IS '景点/目的地信息表';
COMMENT ON COLUMN spots.location IS 'PostGIS地理坐标点，SRID 4326';
COMMENT ON COLUMN spots.crowd_data IS '人流数据: {"current": "low", "hourly": {...}, "weekly": {...}}';
```

#### 2.2.3 行程表 (itineraries)

```sql
CREATE TABLE itineraries (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID NOT NULL REFERENCES users(id),

    -- 基本信息
    title           VARCHAR(200) NOT NULL,
    description     TEXT,
    cover_image     VARCHAR(500),

    -- 时间安排
    start_date      DATE NOT NULL,
    end_date        DATE NOT NULL,
    days_count      INTEGER,

    -- 目的地
    destination     VARCHAR(200),
    destination_location GEOGRAPHY(POINT, 4326),

    -- 行程设置
    budget_level    VARCHAR(20),  -- low, medium, high
    estimated_budget DECIMAL(10,2),
    travel_style    VARCHAR(50)[],  -- 自然, 人文, 美食等
    transportation  VARCHAR(50),  -- self_drive, public, mixed

    -- AI生成相关
    is_ai_generated BOOLEAN DEFAULT FALSE,
    ai_params       JSONB,  -- AI生成时的参数

    -- 统计
    view_count      INTEGER DEFAULT 0,
    favorite_count  INTEGER DEFAULT 0,
    copy_count      INTEGER DEFAULT 0,  -- 被复制的次数

    -- 状态
    status          VARCHAR(20) DEFAULT 'draft',  -- draft, published, archived
    is_public       BOOLEAN DEFAULT FALSE,

    -- 离线数据
    offline_data_url VARCHAR(500),
    offline_size    INTEGER,  -- KB

    -- 时间戳
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    published_at    TIMESTAMP
);

-- 索引
CREATE INDEX idx_itineraries_user ON itineraries(user_id);
CREATE INDEX idx_itineraries_destination ON itineraries(destination);
CREATE INDEX idx_itineraries_dates ON itineraries(start_date, end_date);
CREATE INDEX idx_itineraries_public ON itineraries(is_public, status, published_at DESC);
```

#### 2.2.4 行程项目表 (itinerary_items)

```sql
CREATE TABLE itinerary_items (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- 排序

    start_time      TIME,
    end_time        TIME,

    item_type       VARCHAR(50) NOT NULL,  -- spot, meal, accommodation, transport, note
    custom_title    VARCHAR(200),  -- 自定义标题(非景点时使用)
    custom_content  TEXT,  -- 自定义内容

    -- 费用
    estimated_cost  DECIMAL(10,2),

    -- 备注
    notes           TEXT,

    -- 状态
    status          VARCHAR(20) DEFAULT 'planned',  -- planned, visited, skipped

    -- 时间戳
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_items_itinerary ON itinerary_items(itinerary_id, day_number, order_in_day);
CREATE INDEX idx_items_spot ON itinerary_items(spot_id);
```

## 3. API 设计

### 3.1 API 规范

#### 3.1.1 基础规范

```
Base URL: https://api.hiddengems.app/v1

认证方式: Bearer Token (JWT)
请求格式: JSON
响应格式: JSON
字符编码: UTF-8
```

#### 3.1.2 统一响应格式

```json
// 成功响应
{
    "code": 0,
    "message": "success",
    "data": { ... },
    "meta": {
        "timestamp": "2026-03-21T12:00:00Z",
        "requestId": "req_abc123"
    }
}

// 分页响应
{
    "code": 0,
    "message": "success",
    "data": {
        "items": [...],
        "pagination": {
            "page": 1,
            "pageSize": 20,
            "total": 100,
            "totalPages": 5,
            "hasMore": true
        }
    }
}

// 错误响应
{
    "code": 10001,
    "message": "景点不存在",
    "error": {
        "type": "NotFoundError",
        "details": "Spot with id 'xxx' not found"
    },
    "meta": {
        "timestamp": "2026-03-21T12:00:00Z",
        "requestId": "req_abc123"
    }
}
```

#### 3.1.3 错误码定义

| 错误码范围 | 类型 | 说明 |
|-----------|------|------|
| 0 | 成功 | 请求成功 |
| 10001-10999 | 资源错误 | 资源不存在、已删除等 |
| 20001-20999 | 参数错误 | 参数缺失、格式错误等 |
| 30001-30999 | 认证错误 | 未登录、Token过期等 |
| 40001-40999 | 权限错误 | 无权访问、会员限制等 |
| 50001-50999 | 业务错误 | 业务逻辑错误 |
| 90001-99999 | 系统错误 | 服务器内部错误 |

#### 3.2.1 认证模块 (Auth)

Request:
  {
    "email": "user@example.com",

  }

## 6. 安全设计

### 6.1 认证与授权

```
认证流程:
1. 用户登录 → JWT Token (Access + Refresh)
2. Access Token 有效期: 2小时
3. Refresh Token 有效期: 7天
4. Token 存储在 Redis (支持主动失效)
```

### 6.2 API安全

| 措施 | 说明 |
|------|------|
| HTTPS | 强制HTTPS |
| Rate Limiting | 100 req/min (普通), 1000 req/min (会员) |
| Input Validation | 所有输入参数校验 |
| SQL Injection | 使用参数化查询 |
| XSS | 输出转义 |
| CORS | 白名单域名 |

### 6.3 数据安全

- 敏感数据加密存储
- 密码使用 bcrypt 加密
- PII数据脱敏
- 日志脱敏
